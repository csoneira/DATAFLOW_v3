# About the expected scale factor for the 134, 124 and 13 cases

If the method works fine, the estimated scale factor for impossible trigger types should be 0.

So the scale factor for the 124, 134 and 13 cases should simply be 0: in a corrected by efficiency scenario, these cases simply would not appear at all.

Of course we could simply set 0 as scale factor for those cases, but it's more interesting to see if indeed the scale factor *tends to zero*, which would happen if the method works.

This in interesting because, as 124, 134 and 13 go to zero, if the scale factor is well created, then the 123, for example, would lose the "inefficient part", so it would lose counts that are actually 1234, but at the same time it will win counts from 13 and 12 that are actually 123, and also would keep those true 123 cases, that never cross the fourth plane.

Any impossible trigger combination would fulfill this scale-factor-tend-to-0 issue, but only 134, 124 and 13 are in trigger.
