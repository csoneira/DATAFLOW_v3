## Exact Current Order

1. **Calibration subsample construction**
   - `working_df` is copied to `calibration_work_df`.
   - A strict calibration subsample is selected from raw charge only, not from `clean_tt`.
   - Allowed raw-charge topologies: `1234`, `123`, `124`, `134`, `234`.
   - Builder:
     - `/home/mingo/DATAFLOW_v3/MASTER/STAGES/STAGE_1/EVENT_DATA/STEP_1/TASK_2/script_2_clean_to_cal.py:1962`
   - Selection applied at:
     - `/home/mingo/DATAFLOW_v3/MASTER/STAGES/STAGE_1/EVENT_DATA/STEP_1/TASK_2/script_2_clean_to_cal.py:5450`

2. **Early parameter calculation on the strict subsample**
   - `Q_F` and `Q_B` pedestals are computed with `calibrate_strip_Q_pedestal()`.
   - Pedestal loops start at:
     - `/home/mingo/DATAFLOW_v3/MASTER/STAGES/STAGE_1/EVENT_DATA/STEP_1/TASK_2/script_2_clean_to_cal.py:6995`
   - `T_dif` offsets are computed with `calibrate_strip_T_diff()`.
   - `T_dif` loop starts at:
     - `/home/mingo/DATAFLOW_v3/MASTER/STAGES/STAGE_1/EVENT_DATA/STEP_1/TASK_2/script_2_clean_to_cal.py:7490`

3. **Build strip-component columns**
   - `Q_sum`, `Q_dif`, `T_sum`, `T_dif` are built on `calibration_work_df`.
   - This happens before the sequential calibration-state updates.

4. **Sequential application on the calibration subsample: charge-side first**
   - Since `calibration_dataframe_filtering` is off, the script immediately applies `Q_F/Q_B` to the calibration subsample state:
     - `Q_dif -= (Q_F - Q_B)/2`
     - `Q_sum -= (Q_F + Q_B)/2`
   - Location:
     - `/home/mingo/DATAFLOW_v3/MASTER/STAGES/STAGE_1/EVENT_DATA/STEP_1/TASK_2/script_2_clean_to_cal.py:8110`

5. **Sequential application on the calibration subsample: `T_dif`**
   - `T_dif` is applied to the calibration subsample:
     - `T_dif -= Tdiff_cal`
   - No calibration-dataframe filtering is applied afterwards.
   - Location:
     - `/home/mingo/DATAFLOW_v3/MASTER/STAGES/STAGE_1/EVENT_DATA/STEP_1/TASK_2/script_2_clean_to_cal.py:8174`

6. **Sequential application on the calibration subsample: `T_sum`**
   - `T_sum` is computed from the timing stage and then applied:
     - `T_sum += Tsum_cal`
   - No post-`T_sum` filtering is applied on the calibration dataframe when `calibration_dataframe_filtering: false`.
   - `Tsum_cal` is finalized around:
     - `/home/mingo/DATAFLOW_v3/MASTER/STAGES/STAGE_1/EVENT_DATA/STEP_1/TASK_2/script_2_clean_to_cal.py:9887`

7. **Sequential application on the calibration subsample: `Q_dif` vs `Q_sum`**
   - The calibration dataframe is kept in its already calibrated state.
   - Raw charge components are not restored in this simplified path.
   - The code explicitly states:
     - "Using already pedestal-calibrated Q_sum/Q_dif on the calibration sample before deferred Q_dif-vs-Q_sum front-back calibration."
   - This branch begins at:
     - `/home/mingo/DATAFLOW_v3/MASTER/STAGES/STAGE_1/EVENT_DATA/STEP_1/TASK_2/script_2_clean_to_cal.py:9997`
   - `Q_FB_coeffs` are then fit and applied on that sequentially calibrated subsample.

## Current Conceptual Model

The current Task 2 logic is now much closer to:

1. `build_calibration_subsample`
2. `calculate_QF_QB`
3. `apply_QF_QB_to_calibration_sample`
4. `calculate_Tdif`
5. `apply_Tdif_to_calibration_sample`
6. `calculate_Tsum`
7. `apply_Tsum_to_calibration_sample`
8. `calculate_QFB`
9. `apply_QFB_to_calibration_sample`
10. `apply_all_calibrations_to_working_df`
11. `run_final_unified_filtering`

This is the order that should be kept in mind now, not the older filtered-descendant calibration flow.
