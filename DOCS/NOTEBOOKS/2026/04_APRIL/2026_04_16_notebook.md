## Simplified real-data scale study using `eff_empirical_3`
---

### Prompt / motivation
The aim was to build a very small, purely empirical workflow that reads only real station metadata and produces a corrected rate curve from a single per-plane empirical efficiency. This method is intentionally not based on simulation predictions; it is simply a four-plane correction using `eff_empirical_3 ** 4`.

The specific requirement was to use only `eff_empirical_3`, compute a scale factor from that efficiency, and apply it to the selected rate. The resulting curves and efficiency diagnostics should be captured in two simple plots.

### What was implemented
A new variation was created in:

`MINGO_DICTIONARY_CREATION_AND_TEST/AN_EVEN_EASIER_VARIATION/`

Main files added:

- `step_1_collect_real_data.py`
  - reads real station metadata from the station pipeline outputs for the configured station/date window
  - derives the selected rate column and the four empirical efficiencies without any offender filtering
  - writes `OUTPUTS/FILES/real_data_selected_rate_and_efficiencies.csv`
- `step_2_apply_simplified_scale.py`
  - reads the selected real-data file
  - uses `eff_empirical_3` only as the reference efficiency
  - computes `scale_factor = 1.0 / (eff_empirical_3 ** 4)`
  - applies the factor to `selected_rate_hz`
  - writes `OUTPUTS/FILES/real_data_simplified_scaled_rate.csv`
- `run_all.py`
  - runs step 1 and step 2 in sequence from the same config

Configuration used:

`MINGO_DICTIONARY_CREATION_AND_TEST/AN_EVEN_EASIER_VARIATION/config.json`

Important configuration values:

- `step5.station`: `MINGO01`
- `step5.date_from`: `2024-09-20 00:00:00`
- `step5.date_to`: `2024-09-30 06:10:00`
- `step5.metadata_agg`: `latest`
- `step5.timestamp_column`: `execution_timestamp`
- `trigger_type_selection.task_id`: `5`
- `trigger_type_selection.rate_family`: `four_plane`

### Method
The corrected curve was obtained by the following process:

1. collect the real-data slice for the configured station and date window
2. derive the selected four-plane rate and empirical efficiencies, including `eff_empirical_3`
3. set the reference efficiency to `eff_empirical_3`
4. compute the new scale factor as the inverse of the fourth power of that efficiency:
   `scale_factor = 1.0 / (eff_empirical_3 ** 4)`
5. apply the scale factor to `selected_rate_hz` to obtain `corrected_rate_hz`

This is intentionally simpler than the earlier mean-based variant: it relies entirely on one plane's empirical efficiency instead of averaging `eff_empirical_2` and `eff_empirical_3`, and it is not based on any simulation-derived efficiency model.
