## Task 4 efficiency-vector tuning study from metadata CSVs
---

### Prompt / motivation
The working prompt for this note was:

> I want in simulation tuning a complete study that compares the eff vs X vs Y and vs theta (use the metadata csvs) to determine if the efficiency modelling required tuning or its ok. Especially for some low efficiency regions, i think there is a heavy discrepancy between data and simulation. You study that. Create a new entry in NOTEBOOKS.

The aim was to use the already-produced Task 4 efficiency metadata, not event-by-event files, to decide whether the current efficiency modelling is already acceptable or whether it still needs tuning, with special attention to low-efficiency regions.

### What was implemented
A new ancillary study was added in:

`MASTER/ANCILLARY/SIMULATION_TUNING/EFFICIENCY_VECTOR_TUNING/`

Main script:

`MASTER/ANCILLARY/SIMULATION_TUNING/EFFICIENCY_VECTOR_TUNING/study_efficiency_vector_tuning.py`

It reads `task_4_metadata_efficiency.csv` for the selected simulation and real-data stations, keeps the latest row per `filename_base`, and compares the Task 4 track-based efficiency vectors:

- efficiency vs `x`
- efficiency vs `y`
- efficiency vs `theta`

for planes `1, 2, 3, 4`.

The study writes:

- aggregated per-bin medians and IQRs for simulation and real data
- a real-vs-simulation bin-by-bin summary table
- a plane/axis classification table
- curve plots and discrepancy plots
- a plain-text report with a tuning verdict

To avoid overinterpreting almost-empty bins, the study only compares bins with at least `10` valid files in each group. This is important because some extreme bins can otherwise show artificial `0` or `1` medians just because only a handful of files contribute there.

### Interpretation
The present evidence says the efficiency modelling is not simply “a bit off at the edges”.

What the plots show more specifically is:

- In `x`, planes `1` and `4` are too efficient in the simulation at the outer bins. The real detector is much less efficient there.
- In `y`, especially for planes `2` and `3`, the real data is more efficient than the simulation over a broad range, not only at the most extreme edges.
- In `theta`, planes `2` and `3` also show a systematic positive `REAL - SIM` discrepancy in much of the low-to-mid angle region. The high-theta tail is not the only issue.

So the conclusion is not just “the simulation edge is wrong”. The mismatch is plane-dependent and axis-dependent:

- `x` suggests real-data edge losses stronger than the simulation, especially in planes `1` and `4`;
- `y` suggests the opposite sign for planes `2` and `3`, where the real data looks more efficient than the simulation;
- `theta` confirms that the discrepancy is also visible in angular acceptance, especially for planes `2` and `3`.

This means the current efficiency model should not be treated as validated. It still needs tuning.

### Why this matters for the x-scaling discussion
This study reinforces the warning from the projection-axis work:

- part of the `x` problem is consistent with a real efficiency loss at large `|x|`, especially for planes `1` and `4`;
- but the efficiency mismatch is not purely an `x`-edge effect, because `y` and `theta` also disagree, and sometimes with the opposite sign.

So if one changes the `x` reconstruction scale alone, that may hide only one visible projection of a broader efficiency-modelling problem.

## Baseline-matched answer to the uniform-vs-dependent question
---

### Refined question
The raw average-vs-average comparison above was useful, but it did not answer the narrow physics question cleanly enough.

The refined question is:

> If the simulation curve is first matched to the real-data curve inside a fiducial region, does the rest of the vector still disagree enough to require an explicit dependence on `x` or `y`?

This matters because even a simulation built with uniform per-plane efficiency can show an apparent dependence on `x`, `y`, and `theta` from geometry, acceptance, and the current `cos(theta)` treatment alone.

So the correct test is not just whether two average curves differ. The correct test is whether a **non-position-dependent baseline model**, anchored in a fiducial region, can still explain the full vector.

### Method
The efficiency-vector tuning study was extended so that, for each plane and for each axis `x`, `y`, and `theta`, it now:

1. defines a fiducial baseline region:
   - `|x| <= 100 mm`
   - `|y| <= 100 mm`
   - `theta <= 30 deg`
2. fits a small set of non-position-dependent baseline models using only the fiducial bins:
   - `shift`
   - `scale`
   - `affine`
3. propagates each fitted model to the full vector
4. measures the residual outside the fiducial region
5. decides whether any global baseline is sufficient, or whether explicit dependence is still needed

This is the key distinction:

- `best fiducial model` = the model that fits the fiducial bins best
- `best global baseline` = among models fitted only in the fiducial bins, the one that best explains the **outside-fiducial** vector

The second quantity is the one that actually answers the uniform-vs-dependent question.

### Result
With the current metadata on 2026-04-08, the study now gives the following direct answer:

- `x`: **explicit dependence required**
- `y`: **a global baseline is likely sufficient**
- `theta`: **mixed case; planes 1, 2, 3 are acceptable after baseline matching, but plane 4 still fails**

So the answer is not simply “everything needs x/y dependence” and not simply “everything can be treated as uniform”.

The evidence says:

- `x` still cannot be treated as uniform, because even the best global baseline fitted inside `|x| <= 100 mm` fails badly outside that region for planes `1` and `4`, and remains marginal for planes `2` and `3`.
- `y` is different. Once the simulation is matched in `|y| <= 100 mm`, the best global baseline explains the full `y` vectors well for all four planes.
- `theta` is mostly acceptable after baseline matching except for plane `4`, where the high-angle tail still disagrees enough to require further tuning.

### Interpretation
This refined study narrows the diagnosis considerably.

What it now suggests is:

- the strongest unresolved problem is genuinely on the `x` side, not just a generic efficiency normalization issue;
- the `y` discrepancies seen in the raw average comparison are largely explainable by a global baseline difference once the fiducial region is matched;
- the angular model is not completely broken, but plane `4` still has a residual mismatch in `theta` that is not absorbed by the current baseline treatment.

So, if the question is:

> do we need to add a dependency on `X` and `Y` for the efficiency, or can it be considered uniform in all the plane?

the current answer is:

- **for `x`: no, it cannot be considered uniform**
- **for `y`: yes, at the present evidence level, a non-position-dependent baseline is enough**
- **for `theta`: mostly yes except for plane 4**

### Practical consequence
The strongest tuning priority is now clear:

1. treat the `x` dependence as real and still unresolved;
2. do **not** promote a blanket `y` dependence yet, because the baseline-matched study does not require it;
3. inspect plane `4` angular efficiency separately, because it is the remaining `theta` outlier.

This is a much cleaner conclusion than the original raw comparison, and it is closer to the actual modelling question.
